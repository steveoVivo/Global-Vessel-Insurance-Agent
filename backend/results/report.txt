2. 月次補間による exposure 計算の設計
================================================================================

ユーザー指定の補間式:
  月次増加量 = (来年の隻数 - 今年の隻数) / 12
  月 m (m=1..12) の隻数 = 今年の隻数 + 月次増加量 * (m - 1)

実装 (data_pipeline.py):

  def interpolate_fleet_size(year_sizes, year, month):
      current = year_sizes.get(year, 0)
      next_val = year_sizes.get(year + 1, 0)
      if next_val > 0:
          monthly_increment = (next_val - current) / 12
          return current + monthly_increment * (month - 1)
      return float(current)

  def year_exposure(year_sizes, year):
      # 12ヶ月分の月次隻数の合計 / 12 = 当年の船・年（ship-year）露出
      return sum(interpolate_fleet_size(year_sizes, year, m) for m in range(1, 13)) / 12

Panama での確認例:
  fleet[2011]=8326, fleet[2012]=8299
  月次増加量 = (8299-8326)/12 = -2.25隻/月
  1月: 8326, 7月: 8312, 12月: 8301 (OK)
  2011年の年間露出 = 8313.6 船・年 (旧: 8326隻のみ)

accident_rate の変化:
  旧: 事故数 / 2025年の隻数  (訓練期間の隻数と不一致)
  新: 事故数 / 分析期間の総船・年露出  (訓練/テスト期間で整合)

3. /api/trends への適用
================================================================================

compute_temporal_trends() も同様に更新:

  年 Y のフラグ F の事故率:
    exposure_Y = year_exposure(fleet_sizes, Y)  # 月次補間を使用
    rate_Y     = accidents_in_Y / exposure_Y    # 単位: 事故/船・年

  fleet データがない年の rate は None (= JSON では null)
    → フロントエンドで「データなし」として扱える

Panama 2011-2014 の比較:
               旧 rate (acc/隻)    新 rate (acc/船・年)
  2011             0.006486             0.006495
  2012             0.008435             0.008434
  2013             0.008795             0.008862
  2014             0.007228             0.007294

  → 値は近いが、2013年以降で差が出ている。
    これは 2013→2014 で隻数が減少しており、月次補間が年中の増減を正確に反映しているため。

4. 評価結果の推移
================================================================================

                       ベースライン   正規化改善後   月次補間後(最終)
  accident_rate の計算  ÷2025隻数     ÷2025隻数      ÷露出(船・年)
  正規化方式            minmax        minmax_capped  minmax_capped
  ship_type_risk        linear        log1p          log1p
  Spearman r            -0.108        +0.144         +0.097
  p-value               0.5688        0.4479         0.6109
  評価                  WEAK          WEAK           WEAK

全てのステップで r < 0.3 は維持されているが、方向性は改善している:
  ベースライン: r = -0.108  (逆相関 — スコアが高いほど実際に安全な傾向)
  最終:         r = +0.097  (正相関 — スコアが高いほど事故率も高い傾向)

正規化改善のステップで r = +0.144 に上昇したが、月次補間後に r = +0.097 に若干低下した。
これは月次補間により test_rate の計算が変わった (÷2025隻数 → ÷2021-2025露出) ため、
訓練スコアとの相関構造が変化したことによる。内部整合性は向上している。


================================================================================
5. r が弱い主な原因
================================================================================

  (a) Paris MoU の影響が大きすぎる
      flagの Paris MoU category (white/grey/black) は静的データ (2024年のみ)。
      訓練期間 (2011-2020) の安全評価を反映していない。
      また、Viet Nam・Belize は Paris MoU で高スコアだが、
      実際の 2021-2025 の事故率は低い。

  (b) 等重みの問題
      4成分 (accident_rate, severity, ship_type, flag_safety) に均等重み 0.25。
      事故率の予測力が低い場合、他の成分がノイズを追加している可能性がある。

  (c) COVID-19 の影響
      2020年の運航急減により訓練期間の末尾データが特異的になっている。
      訓練期間を 2011-2019 にすることで改善する可能性がある。

  (d) 船舶数データの制限
      2025年の年次スナップショットが最新値。2025年以降の補間ができない。

  (e) 大幅な変動がある国
      大国 (UK, Malta) は 2011-2020 は比較的低スコアだが、
      2021-2025 で事故率が高い → 訓練データだけでは予測困難。

大幅に追加されたため、何が追加され、どのように評価され、現時点で何が効いて

2. 全体として何が変わったか
--------------------------------------------------------------------------------

今回の主な変更は、「単純な事故数・重大度・船種・Paris MoU の4要素」から、
事故の性質、事故原因の多様性、時系列トレンド、船隊規模の変化、Paris MoU の
連続値情報まで含む、多変量リスクスコアへ拡張した点である。

現行 DEFAULT_WEIGHTS:
  accident_rate:       0.40
  event_entropy:       0.25
  trend:               0.20
  investigation:       0.15
  その他の拡張変数:    0.00

つまり、API上は15個の変数を受け取れるが、デフォルトのリスクスコアで実際に
使っているのは以下の4つである。

  - accident_rate
  - event_entropy
  - trend
  - investigation

これらは単体評価で Spearman r >= 0.4 かつ p < 0.05 だった変数として選ばれて
いる。ただし DEFAULT_WEIGHTS はテストデータを見た後に調整された重みなので、
evaluate_risk_score.py では「参考値 / biased」として扱われている。

4. 追加・拡張された変数の意味
--------------------------------------------------------------------------------

以下は build_merged_rows() が最終的に持つ主な説明変数である。
各変数は最終的に *_norm として 0-1 に正規化され、重み付き平均で risk_score に
入る。値が大きいほどリスクが高い、という向きに揃えられている。

(1) accident_rate / accident_rate_norm
  意味:
    事故数を、その期間の船・年 exposure で割った事故率。

  計算:
    accident_count / total_ship_year_exposure

  ポイント:
    以前のように 2025年の船舶数だけで割るのではなく、2011-2025年の年次船舶数を
    月次線形補間して exposure を作っている。船隊が増減している国でも、分析期間
    に対応した事故率になる。

(2) severity_risk_norm
  意味:
    事故の重大度リスク。

  計算:
    avg_severity_risk と vsmc_rate を 0.6 : 0.4 でブレンド。

  内訳:
    avg_severity_risk:
      Marine incident = 0.25
      Marine casualty = 0.65
      Very serious marine casualty = 1.0
      として平均したもの。

    vsmc_rate:
      Very serious marine casualty の割合。

(3) ship_type_risk_norm
  意味:
    事故に出てくる船種の頻度リスク。

  計算:
    船種ごとの事故出現数を log1p で圧縮し、各旗国事故の平均を取る。

  ポイント:
    事故数が極端に多い船種だけがスコアを支配しないよう、対数変換を使っている。

(4) flag_safety_risk_norm
  意味:
    Paris MoU の detention rate に基づく旗国安全リスク。

  計算:
    total_detentions / total_inspections

  ポイント:
    以前の White/Grey/Black の3値ではなく、検査数と拘留数から連続的な detention
    rate を作っている。DEFAULT_WEIGHTS では 0.00 なので、現行デフォルトスコア
    には入っていない。

(5) trend_slope_norm
  意味:
    年次事故率が悪化しているかどうか。

  計算:
    年ごとの exposure-weighted accident rate に対する OLS slope。

  解釈:
    slope > 0 なら事故率が上昇傾向でリスク増。
    slope < 0 なら改善傾向なので、正規化前に 0 へクリップされる。

(6) multi_ship_rate_norm
  意味:
    複数船が関与した事故の割合。

  計算:
    Number of ships involved >= 2 の事故数 / 全事故数。

  解釈:
    衝突・接触など、相手船や交通環境を含む複合的な事故傾向を表す。

(7) collision_rate_norm
  意味:
    事故種別が collision 系である割合。

  計算:
    Casualty event の先頭カテゴリが collision で始まる事故数 / 全事故数。

(8) open_sea_rate_norm
  意味:
    Open sea で起きた事故の割合。

  計算:
    Location が open sea の事故数 / Location が既知の事故数。

  解釈:
    港内・沿岸ではなく外洋運航中のリスク傾向を表す。

(9) event_entropy_norm
  意味:
    事故原因カテゴリの多様性。

  計算:
    Casualty event の上位カテゴリ分布に Shannon entropy を適用。

  解釈:
    値が高いほど、事故原因が一種類に集中せず、多様な原因で事故が起きている。
    コード上では「単一問題ではなく、よりシステム的・多面的なリスク」として
    扱われている。

(10) investigation_rate_norm
  意味:
    事故1件あたりの調査報告数。

  計算:
    Number of investigation reports の合計 / 事故数。

  注意:
    コメント上は当初「高いほど安全」とも読めるが、実装では反転せず、
    高いほどリスク増として扱っている。理由は、単体評価で将来事故率と正に相関
    したためである。重大事故が多い国ほど調査報告も多くなりやすい、という代理
    変数として働いている可能性がある。

(11) solas_noncompliance_norm
  意味:
    SOLAS 非準拠船が関与した事故の割合。

  計算:
    SOLAS status に ": N" を含む事故数 / SOLAS status が既知の事故数。

(12) excess_factor_norm
  意味:
    Paris MoU の excess factor 平均。

  計算:
    分析期間内の excess_factor の平均。

  解釈:
    正の値が大きいほど、期待値より悪い Port State Control 成績を示す。
    負の値は改善・安全側なので 0 にクリップされる。

(13) excess_factor_trend_norm
  意味:
    Paris MoU excess factor の時系列悪化傾向。

  計算:
    年ごとの excess_factor に対する OLS slope。

  解釈:
    正なら Paris MoU 成績が悪化傾向。負なら改善傾向として 0 にクリップ。

(14) fleet_growth_norm
  意味:
    船隊規模の成長率。

  計算:
    (period_end_fleet - period_start_fleet) / period_start_fleet

  解釈:
    急拡大するレジストリは監督体制が追いつかない可能性がある、という仮説。
    負の成長はこの観点ではリスク増と見なさず、0 にクリップされる。

(15) fleet_volatility_norm
  意味:
    船隊規模の年次変動の激しさ。

  計算:
    年次変化量の RMSE / 期間平均船隊規模。

  解釈:
    登録・離脱が激しい flag-of-convenience 的な不安定性を表す代理変数。


5. 正規化・スコア化の設計
--------------------------------------------------------------------------------

各変数は minmax_capped() によって 0-1 に正規化される。

特徴:
  - 非ゼロ値の95パーセンタイルを上限として使う
  - 極端な外れ値がある場合でも、他の国の差が 0 付近に潰れにくい
  - 上限を超えた値は 1.0 にクリップされる

符号付きの変数:
  - trend_slope
  - excess_factor_avg
  - excess_factor_trend
  - fleet_growth_rate

これらは「負の値 = 改善または安全側」と解釈し、正規化前に max(value, 0) で
クリップされる。

risk_score:
  risk_score = sum(weight_i * normalized_factor_i) / sum(weights)

APIでは w1-w15 のクエリパラメータで重みを上書きできる。

  w1  accident_rate
  w2  severity
  w3  ship_type
  w4  flag_safety
  w5  trend
  w6  multi_ship
  w7  collision
  w8  open_sea
  w9  event_entropy
  w10 investigation
  w11 solas_noncompliance
  w12 excess_factor
  w13 excess_factor_trend
  w14 fleet_growth
  w15 fleet_volatility


6. 評価方法
--------------------------------------------------------------------------------

evaluate_risk_score.py の評価設計:

  訓練期間:
    2011-2020

  テスト期間:
    2021-2025

  対象:
    2025年船隊規模の上位30旗国

  評価指標:
    訓練期間から作った risk_score と、テスト期間の実測 accident_rate
    (事故数 / ship-year exposure) の Spearman rank correlation。

  Spearman r の読み方:
    +1 に近いほど「高リスクと予測した国ほど、将来の実事故率も高い」。
    0 付近なら順位予測としてほぼ無相関。
    負なら逆方向。

重要:
  evaluate_risk_score.py の PRIMARY EVALUATION は DEFAULT_WEIGHTS ではなく、
  EVAL_WEIGHTS を使う。これは「テスト結果を見て調整した重み」で評価を水増し
  しないための設計である。

PRIMARY EVAL_WEIGHTS:
  accident_rate: 0.20
  event_entropy: 0.20
  trend: 0.20
  investigation: 0.20
  flag_safety: 0.20
  その他: 0.00


7. 現行コードの評価結果
--------------------------------------------------------------------------------

PRIMARY EVALUATION:

  Spearman r = 0.558
  p-value    = 0.0014
  判定       = GOOD

解釈:
  2011-2020年の情報から作ったリスクスコアは、2021-2025年の実事故率順位を
  ある程度よく予測できている。p-value も 0.0014 と小さいため、上位30旗国に
  限った評価では統計的にも意味のある正相関と見てよい。

DEFAULT_WEIGHTS:

  Spearman r = 0.599
  p-value    = 0.0005

解釈:
  現行デフォルト重みは PRIMARY EVALUATION よりさらに高い相関を出している。
  ただし、この重みはテストデータを見た後に調整された可能性があるため、
  unbiased な評価値としては PRIMARY EVALUATION の r = 0.558 を採用するのが
  妥当である。

--------------------------------------------------------------------------------

  accident_rate              r = +0.674   p = 0.0000
  event_entropy              r = +0.477   p = 0.0078
  trend                      r = +0.458   p = 0.0109
  investigation              r = +0.439   p = 0.0153
  flag_safety                r = +0.159   p = 0.4026
  severity                   r = -0.181   p = 0.3388
  ship_type                  r = +0.042   p = 0.8272
  multi_ship                 r = -0.079   p = 0.6782
  collision                  r = +0.134   p = 0.4807
  open_sea                   r = +0.053   p = 0.7790
  solas_noncompliance        r = -0.246   p = 0.1905
  excess_factor              r = NaN      p = NaN
  excess_factor_trend        r = NaN      p = NaN
  fleet_growth               r = +0.097   p = 0.6117
  fleet_volatility           r = +0.145   p = 0.4437

主な発見:

  (A) accident_rate が最も強い
      r = +0.674。過去の exposure-adjusted 事故率は、将来事故率の強い予測因子。

  (B) event_entropy, trend, investigation は有意
      3つとも r >= +0.439 かつ p < 0.05。
      事故原因の多様性、事故率の悪化傾向、調査報告率は追加ファクターとして
      意味がある。

  (C) Paris MoU detention rate は弱い
      flag_safety は r = +0.159 で p = 0.4026。
      PRIMARY EVALUATION には等重みで入っているが、単体では有意ではない。

  (D) severity と solas_noncompliance は逆相関
      今回の評価条件では、重大度やSOLAS非準拠率は将来事故率と正に相関して
      いない。少なくともデフォルト重みから外す判断は妥当。

  (E) excess_factor 系は NaN
      上位30旗国・訓練期間の評価条件では、正規化後の入力が定数に近くなり、
      Spearman相関が定義できなかった。現状では予測因子として評価不能。


9. 感度分析
--------------------------------------------------------------------------------

evaluate_risk_score.py の出力に含まれる主な組み合わせ:

  Accident rate only                         r = +0.674
  Detention rate only                        r = +0.159
  Trend slope only                           r = +0.458
  Severity only                              r = -0.181
  Ship type only                             r = +0.042
  Acc + Trend                                r = +0.789
  Acc + Detention + Trend                    r = +0.817
  Equal 3 (acc, det, trend)                  r = +0.831
  Current DEFAULT_WEIGHTS (biased)           r = +0.599

読み方:
  事故率とトレンドを組み合わせると r = +0.789 まで上がる。
  さらに detention rate を加えると r = +0.817、3つを等重みにすると r = +0.831。
  ただし、これらの組み合わせは「テスト結果を見た後の参考分析」であり、
  unbiased な本評価としては扱わない。

実務的には、事故率・トレンド・事故原因多様性・調査報告率が説明しやすく、
かつテスト独立性を主張しやすい。detention rate は組み合わせでは効く可能性が
あるが、単体では弱いので、採用する場合は根拠説明が必要。

12. 結論
--------------------------------------------------------------------------------

今回の変更により、バックエンドは単なる事故率スコアから、以下を含む多面的な
リスクモデルへ拡張された。

  - exposure-adjusted accident rate
  - 年次事故率トレンド
  - 事故原因カテゴリの多様性
  - 調査報告率
  - 事故重大度
  - 船種リスク
  - 複数船事故・衝突・外洋事故
  - SOLAS非準拠
  - Paris MoU detention / excess factor
  - 船隊成長率・船隊変動性

現時点で、独立性を意識した PRIMARY EVALUATION は r = +0.558, p = 0.0014 であり、
「将来事故率の順位予測として意味のある信号がある」と言える。

一方、単体で強いのは accident_rate, event_entropy, trend, investigation であり、
その他の変数は現状では説明用・実験用の候補として残されている位置づけである。
